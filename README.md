
## Description
Ce projet consiste à développer une application de gestion de catalogue de produits permettant de créer, consulter, mettre à jour, supprimer et interagir avec des produits (like/dislike, requêtes par sous-ensemble).  
Le projet est mené selon une méthodologie Agile, avec un suivi clair de l’avancement via ZenHub et un Burndown Chart.

## Objectifs du projet
- Gérer un catalogue de produits (CRUD)
- Rechercher et filtrer un sous-ensemble de produits
- Ajouter des interactions utilisateur (Like / Dislike)
- Héberger l’application dans le cloud
- Mettre en place une automatisation du déploiement
- Visualiser l’avancement et la vélocité de l’équipe

## Fonctionnalités principales
- Création d’un produit
- Consultation d’un produit
- Mise à jour d’un produit
- Suppression d’un produit
- Like / Dislike d’un produit
- Recherche et filtrage des produits

## Gestion de projet (Agile)
Le projet est organisé à l’aide de ZenHub, avec les colonnes suivantes :
- New Issues
- Icebox
- Product Backlog
- Sprint Backlog
- In Progress
- Review / QA
- Done

### Burndown Chart
Un Burndown Chart a été ajouté afin de :
- Suivre l’avancement du sprint
- Visualiser le travail restant
- Vérifier le respect des délais

## Outils utilisés
- ZenHub pour la gestion du backlog et des sprints
- Burndown Chart pour le suivi de la progression
- Git / GitHub pour la gestion du code source
- Hébergement cloud pour le déploiement de l’application
- CI/CD pour l’automatisation des déploiements

## Déploiement
L’application est hébergée dans le cloud avec un pipeline d’automatisation permettant :
- Le déploiement automatique des nouvelles versions
- La réduction des erreurs humaines
- Une mise en production plus rapide
